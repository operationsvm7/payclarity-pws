## Architecture

**PayClarity PWS** is a commission management SPA deployed on **Vercel**, using TanStack Router (React 19, no SSR) with **Supabase** (PostgreSQL + Auth) as the backend.

> The project was originally built in Lovable. It was migrated in June 2026 to a standard Vite SPA by removing `@lovable.dev/vite-tanstack-config`, `@cloudflare/vite-plugin`, `@tanstack/react-start`, and `nitro`. The entry point is now `index.html` → `src/main.tsx`. Deployment target is Vercel (`vercel.json` at root).

### Routing & Entry Points

- `index.html` + `src/main.tsx` — SPA bootstrap; creates the TanStack Router instance and mounts React
- `src/routes/__root.tsx` — root layout; `beforeLoad` redirects unauthenticated users to `/login`; wraps the outlet in `QueryClientProvider` + `AuthProvider`
- `src/routes/index.tsx` — mounts `<CommissionTool />`, the main app shell (requires auth)
- `src/routes/login.tsx` + `src/routes/register.tsx` — public auth pages (bypassed if already signed in)
- `routeTree.gen.ts` — auto-generated by TanStack Router plugin on `dev`/`build`; do not edit manually
- `src/router.tsx` — exports `getRouter(queryClient)` used by `main.tsx`

### Authentication

Auth is handled client-side via Supabase + `src/lib/auth-context.tsx`:

- `AuthProvider` / `useAuth()` — provides `{ user, profile, loading, signOut, refreshProfile }`
- **Profile** fields: `id`, `email`, `full_name`, `role` (`admin|rep|accountant|null`), `status` (`pending|active|rejected`)
- **Register flow**: user provides email + password + company invite code → code verified server-side via `verify_invite_code()` RPC → first user auto-becomes admin; all others start as `pending`
- **Admin approval**: Settings → User Management tab (`UserManagementPanel.tsx`) — admin assigns role and approves; pending users cannot enter the app
- **Role sync**: `CommissionTool.tsx` reads `useAuth().profile.role` and syncs it into the Zustand store via `useEffect`

Run `supabase/migrations/001_auth_setup.sql` in the Supabase SQL Editor to create the `profiles` and `company_config` tables with RLS and triggers.

### State Management

Global app state lives in the Zustand store (`commission-store.ts`), persisted to localStorage. On first launch with an empty store, demo data is auto-seeded. The user's role comes from Supabase (`profiles.role`) and is synced into Zustand; data (agents, invoices, etc.) still lives in localStorage.

### Key Data Model Concepts

- **Agent** — has a role (`admin|rep|accountant`), commission level, sponsor (for downline hierarchy), and wallet
- **Invoice** — tracks sales amount, approval amount, product cost, finance company fees, charges/credits, and CCPF
- **Split** — distributes commission percentages across multiple participants on an invoice
- **Commission Plan** — volume-based tiered rates; supports overrides for downline agents
- **Ledger Entry** — individual wallet transaction (commission, override, advance, deduction)

### Supabase Integration

- `src/integrations/supabase/client.ts` — browser-side Supabase client (uses `VITE_SUPABASE_URL` + `VITE_SUPABASE_PUBLISHABLE_KEY`)
- `src/integrations/supabase/types.ts` — TypeScript types matching the DB schema (`profiles`, `company_config`, RPC functions)
- `src/integrations/supabase/client.server.ts` — server-side admin client stub (unused in SPA mode)

Credentials: copy `.env.example` → `.env` and fill in values from your Supabase project dashboard.

### Styling

Tailwind CSS 4 with shadcn/ui. Component variants use `class-variance-authority`; classes are merged with `clsx` + `tailwind-merge` (exposed as `cn()` from `src/lib/utils.ts`). The `@/*` path alias resolves to `src/`. Fonts (Inter, Space Grotesk, JetBrains Mono) are loaded via Google Fonts in `index.html`.

### Deployment (Vercel)

- `vercel.json` at root — sets build command, output dir (`dist`), and SPA rewrites
- Add `VITE_SUPABASE_URL` and `VITE_SUPABASE_PUBLISHABLE_KEY` as environment variables in the Vercel project dashboard
- `wrangler.jsonc` is kept for reference only; the Cloudflare Workers deployment is no longer active
